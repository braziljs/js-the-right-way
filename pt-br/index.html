INICIANDO

SOBRE

Criado pela Netscape em 1995 como uma extensão do HTML para o Navegador Netscape 2.0, Javascript teve como função principal a manipulação de documentos HTML e validação de formulários. Antes de ganhar este nome tão famoso hoje em dia, Javascript foi chamado de Mocha. Quando enviado pela primeira vez em versões beta, foi oficialmente chamado de LiveScript e quando foi lançado oficialmente pela Sun Microsystems foi batizado pelo nome qual é conhecido hoje. Por causa desses nome similares, as pessoas confudem Javascript com Java. Embora ambos tenham a estrutura léxical de programação, eles não são a mesma lingua. Diferente de C, C# e Java, Javascript é uma linguagem interpretada. Isso significa que é necessário um "intérprete". No caso do Javascript, o "intérprete" é o navegador.

VERSÃO ATUAL

O norma Javascript é ECMAScript. A partir de 2012, todos os navegadores modernos suportam totalmente o ECMAScript 5.1. Navegadores antigos suportam pelo menos a versão ECMAScript 3. Uma grande revisão do padrão 6 está sendo trabalhado.

Uma boa referência para versões, referências e novidades sobre Javascript podem ser achadas em Mozilla Developer Network.

O DOM

O Document Object Model (DOM) é uma API para documentos HTML e XML. Que permite que você modifique o seu conteúdo e apresentação visual usando uma linguagem de script como Javascript. Veja mais no Mozilla Developer Network - DOM

AS BOAS PARTES

Orientação a Objetos

Javascript tem uma forte orintação a objetos. Apesar de alguns debates ocorrerem devido às diferenças em Javascript orienta a objetos em comparação com outras linguagens.

Fonte: Introduction to Object-Oriented JavaScript

Funções anônimas

Funções anônimas são funções que são declaradas dinamicamente em tempo de execução. Elas são chamadas de funções anônimas porque elas não recebem um nome da mesma forma que as funções normais.

Fonte: JavaScript anonymous functions

Funções como Objetos de Primeira Classe

Funções em Javascript são objetos de primeira classe. Isso significa que as funções JavaScript são apenas um tipo especial de objeto que pode fazer todas as coisas que os objetos comuns podem fazer.

Fonte: Functions are first class objects in JavaScript

Fracamente Tipada

Para muitos desenvolvedores front-end, Javascript foi seu primeiro gosto como uma linguagem de script e/ou interpretação. Para esses desenvolvedores os conceitos e implicações de váriaveis fracamente tipadas pode ser uma segunda natureza. No entanto, o crescimento explosivo de aplicações modernas resultou em um número crescente de desenvolvedores de back-end que tiveram que mergulhar os pés na piscina de tecnologias no lado cliente. Muito desses desenvolvedores estão vindo de linguagens fortemente tipadas como C# ou Java, e eles não são familiares com a liberdade e o potencial que envolve o trabalho com váriaveis fracamente tipadas.

Fonte: Understanding Loose Typing in JavaScript

Escopo e Hoisting

Escopo: em Javascript, as nossas funções são de fato os delimitadores de escopo para váriaveis. Significa que bloqueia eventuais loops e condicionais (como if, for, while, switch e try) não delimitam o escopo. Ao contrário da maioria das outras linguagens. Portanto, esses blocos irão compartilhar o mesmo escopo que as funções que os contém. Desta forma, pode ser perigoso declarar váriaveis em blocos pode parecer que a váriavel pertence apenas a este bloco.

Hoisting: Em tempo de execução, todas as declarações de váriaveis e funções são movidas para o inicio de cada função(no seu escopo) - isto é conhecido como hoisting. Dito assim, é uma boa prática declarar todas as váriaves completamente na primeira linha, a fim de evitar falsas expectativas com a váriavel que tem declaração mais tarde, mas aconteceu para manter um valor antes - este é um problema comum para programadores proveniente de linguagens com escopo de bloco.

Fonte: JavaScript Scoping and Hoisting

Função Binding

Função binding é provavelmente a menor de suas preocupações quando começando com Javascript, mas quando você percebe que precisa de uma solução para o problema de como manter o contexto deste dentro de outra função, então você pode perceber que o que você realmente precisa é de Function.prototype.bind().

Fonte: Understanding JavaScript’s Function.prototype.bind

Funções Closure

Closures são funções que se referem a váriaveis independentes(livres). Em outras palavras, as funções definidas em uma closure "lembra-se" ao ambiente na qual ele foi criado. É um conceito importante para compreender como ele pode ser útil durante o desenvolvimento, como emular metódos privados. E também pode aprender a evitar erros comuns, como criar closures em loops.

Fonte: MDN - Closures

Modo Estrito

Modo estrito do ECMAScript 5 é uma forma de optar por uma variante de Javascript. Modo estrito não é apenas um subconjunto: intencionalmente tem diferentes semânticas de código normal. Navegadores que não suportam o modo estrito irão rodar o modo estrito diferentes dos navegadores que suportam, então não dependem do modo estrito, sem recurso de teste de suporte para os aspectos relevantes para o modo estrito. Código de modo estrito e não estrito podem coexistir, de modo que os scripts podem optar o modo estrito de forma incremental.

Fonte: MDN - Strict mode

Funções de chamadas imediatas(Immediately-Invoked Function Expression(IIFE))

Uma função de chamada imediata é um padrão que produz um escopo léxico usando a função de definição do âmbito de Javascript. Funções imeditas podem ser usadas para evitar elevação de váriavel de dentro de blocos, proteger contra a poluição do ambiente global e simultaneamente permitir o acesso público aos métodos, mantendo a privacidade para as váriaveis definidas dentro da função.

Esse padrão tem sido referida como uma função anônima auto-executável, mas o @cowboy (Ben Alman) introduziu o termo IIFE como um termo mais preciso semanticamente para o padrão.

Fonte: Immediately-Invoked Function Expression (IIFE)

Dojo é livre, kit de ferramentas aberto para construção de aplicações de alta performance. Os patrocinadores do projeto incluem IBM e o SitePen.

  • Cappuccino

Cappuccino é um framework de código aberto que facilita a criação de aplicativos de desktop que são executados em um navegador web.

JavaScriptMVC é um framework código aberto que contém as melhores ideias em desenvolvimento jQuery.

  • Meteor

Meteor é uma plataforma de código aberto para a construção de aplicações web de alta qualidade em uma fração de tempo, se você é um desenvolvedor especialista ou apenas começando.

Motim é incrivelmente rápido, poderoso e com uma pequena biblioteca no lado cliente (MV *) para a construção de aplicações web de grande escala. CanJS é um framework Javascript que torna o desenvolvimento de aplicações complexas simples e rápida. Fácil de aprender, pequeno, despretensioso e de sua estrutura de aplicação, mas com características modernas, como as tags personalizadas e de 2 vias de ligação.

Motores de Jogos


  • MelonJS

MelonJS é livre, leve e em HTML5. Esse motor integra o famoso formato de telha, permitindo que seja facilmente usado.

  • ImpactJS

ImpactJS é um dos mais testados e verdadeiro motor de jogos em HTML5 com a versão inicial por volta de 2010. É bem mantido e atualizado e tem uma boa comunidade para apoiá-lo. Existe uma abundância de documentção - tem até dois livros sobre o assunto de criação de jogos no motor.

  • LimeJS

LimeJS é uma estrutura de jogo HTML5 para a criação de jogos rápidos, pela experiência nativa e para todos os touchscreens modernos e navegadores desktop.

  • Crafty

Crafty é um motor de jogo que volta ao final de 2010. Crafty tornal muito facil para começar a criar jogos em Javascript.

  • Cocos2d-HTML5

Cocos2d-html5 é um framework de código aberto, distribuido sob a licença MIT. É uma versão HTML5 do projeto Cocos2d-x. O foco do desenvolvimento do Cocos2d-html5 é em torno de fazer plataforma cruzadas Cocos2d entre navegadores e aplicativos nativo.

  • Phaser

Phaser é fortemente baseada em Flixel. É mantido por Richard Davey (Photon Storm), que tem sido muito ativo na comunidade HTML5 por anos.

  • Goo

Goo é um motor de jogo em Javascript inteiramente construido em WebGL/HTML5.

  • LycheeJS

LycheeJS é uma biblioteca de jogo Javascript que oferece uma solução completa para prototipação e desenvolvimento em HTML5 Canvas, WebGL ou OpenGL(ES) nativa com base em jogos que estão nos navegadores modernos ou ambientes nativos.

  • Turbolenz

Turbulenz é apoiado por 5 milhões de dólares em financiamento de risco. O motor é muito polido e focado principalmente em jogos HTML5 de alta qualidade. Parece ter pouco suporte para dispositivos móveis, mas eles vão logo pega-los quando todos os dispositivos móveis suportarem WebGL.

  • Quintus

Quintus é um motor de jogo em HTML5 projetada para ser modular e leve, com uma sintaxe Javascript amigável.

  • KiwiJS

Kiwi.js é de código aberto, divertida e amigável motor de jogo em HTML5. Muitas pessoas o chamam de Wordpress dos motores de jogos em HTML5.

  • PandaJS

Panda.js é um motor de jogo em HTML5 para dispositivos mobiles e desktops com renderização em Canvas e WebGL.
